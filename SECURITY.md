# Política de segurança

## Versões suportadas

Este repositório é principalmente **educativo** (documentação e exemplos). Não há política formal de suporte a versões de linguagem ou pacotes além do que estiver documentado em `src/python/pyproject.toml` e nos projetos .NET em `src/csharp/`.

## Reportar uma vulnerabilidade

Se você identificar um problema de segurança **neste repositório** (por exemplo, no código de exemplo, scripts ou configuração de CI):

1. **Abra uma issue** com o título prefixado `[SECURITY]` descrevendo o problema de forma responsável, **sem** explorar publicamente antes de haver correção ou orientação, ou
2. Use os canais privados do GitHub (**Security advisories**), se disponíveis na organização/conta do repositório, para divulgação confidencial.

Inclua, quando possível: passos para reproduzir, impacto estimado e sugestão de mitigação.

## Expectativa de resposta

Os mantenedores farão o possível para avaliar relatórios em tempo razoável. Este é um projeto de estudo; não há SLA formal.

## Escopo

Problemas em **dependências de terceiros** ou em **ferramentas externas** devem ser reportados aos respectivos projetos oficiais.
